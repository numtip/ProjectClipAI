# Render Log Standard

> Each project must maintain a render log in `projects/<project-id>/reports/render-log.md`.
> This is autogenerated by `render-batch-ltx.bat`.

---

## Required Fields

Each render log entry must include:

| Field | Description | Example |
|-------|-------------|---------|
| `project_id` | Project identifier | `project-001` |
| `source_image` | I2V source image used | `ComfyUI/input/first-render-001.png` |
| `workflow` | ComfyUI workflow JSON | `first-video-002.json` |
| `shot_id` | Shot identifier | `Shot01` |
| `prompt` | Positive prompt used | `cinematic drone flight...` |
| `start_time` | Render start time | `10:15:23` |
| `end_time` | Render end time | `10:17:18` |
| `render_duration` | Time to render (seconds) | `115.3` |
| `output_file` | Generated MP4 filename | `Shot01.mp4` |
| `file_size` | Output file size | `1.86 MB` |
| `validation` | ffprobe check result | `PASS` |
| `notes` | Any anomalies or observations | `First shot warm-up, slightly longer` |

---

## Log Format

### File Header

```markdown
# Render Log

**Project:** project-001
**Date:** 2026-06-18
**Workflow:** first-video-002.json
**Source Image:** ComfyUI/input/first-render-001.png
**Model:** LTX Video 2B v0.9.1
**Encoder:** T5XXL fp8

---

## Shot Summary

| Shot | Seed | Render Time | File Size | Validation | Notes |
|------|------|-------------|-----------|------------|-------|
```

```markdown
### Shot01

```

## Validation Criteria

After rendering each shot, validate with:

```powershell
ffprobe -v error -show_entries format=duration,size,bit_rate:stream=codec_name,width,height,r_frame_rate -of default=noprint_wrappers=1 <file>
```

**Pass criteria:**
- `codec_name`: h264
- `width`: 768
- `height`: 512
- `r_frame_rate`: 24/1
- `duration`: 5.0-5.2s
- `size`: > 500 KB

**Fail indicators:**
- File size < 100 KB (likely corrupt or empty)
- Duration < 4.5s or > 6.0s
- Any ffprobe error
- Codec not h264

